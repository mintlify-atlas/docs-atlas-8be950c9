---
title: "Permissions"
description: "Manage what players and groups can do on your server using EssentialsX's permission nodes."
---

EssentialsX uses your server's permissions plugin to control access to every command and feature. Each capability is guarded by a named permission node in the format `essentials.<feature>`. You assign these nodes to players or groups through a permissions plugin such as [LuckPerms](https://luckperms.net/).

## The `essentials.*` wildcard

Operators receive the `essentials.*` permission by default. This wildcard grants every EssentialsX permission, including all subcommands and features. It is equivalent to granting all individual nodes listed below.

```
essentials.*
```

<Warning>
Granting `essentials.*` to non-operator groups gives those players access to admin commands such as `/eco`, `/give`, `/ban`, and `/vanish`. Only grant the wildcard to trusted staff groups.
</Warning>

## Wildcard patterns

EssentialsX supports wildcard inheritance for permission groups that share a common prefix. Use these to grant broad access without listing every node individually.

| Wildcard | What it grants |
|---|---|
| `essentials.home.*` | All `/home` sub-permissions |
| `essentials.warp.*` | All `/warp` sub-permissions (including per-warp nodes) |
| `essentials.sethome.*` | All `/sethome` sub-permissions including multiple homes |
| `essentials.kits.*` | Access to all defined kits |
| `essentials.signs.*` | All sign creation, use, and break permissions |
| `essentials.gamemode.*` | All gamemode sub-permissions |

<Tip>
If your permissions plugin supports Bukkit-style wildcards, set `use-bukkit-permissions: true` in `config.yml`. When enabled, EssentialsX respects wildcard inheritance natively without requiring Vault.
</Tip>

## `essentials.gamemode.*` structure

The `essentials.gamemode.*` node is defined in `plugin.yml` with explicit children, which means granting the wildcard automatically grants the following:

```yaml
essentials.gamemode.*:
  children:
    essentials.gamemode: true        # Access to the /gamemode command
    essentials.gamemode.others: true # Change another player's gamemode
    essentials.gamemode.all: true    # Access to all four gamemodes
```

You can restrict access to specific gamemodes by granting individual nodes instead:

| Node | Grants |
|---|---|
| `essentials.gamemode.survival` | `/gamemode survival` |
| `essentials.gamemode.creative` | `/gamemode creative` |
| `essentials.gamemode.adventure` | `/gamemode adventure` |
| `essentials.gamemode.spectator` | `/gamemode spectator` |

## Granting permissions with LuckPerms

Use the LuckPerms `/lp` command to assign permission nodes to groups or players.

**Grant a permission to a group:**

```bash
lp group default permission set essentials.home true
lp group default permission set essentials.sethome true
lp group default permission set essentials.warp true
lp group default permission set essentials.tpa true
```

**Grant a wildcard to a moderator group:**

```bash
lp group moderator permission set essentials.ban true
lp group moderator permission set essentials.kick true
lp group moderator permission set essentials.mute true
lp group moderator permission set essentials.vanish true
lp group moderator permission set essentials.socialspy true
```

**Grant all EssentialsX permissions to an admin group:**

```bash
lp group admin permission set essentials.* true
```

**Deny a specific permission (override):**

```bash
lp group default permission set essentials.fly false
```

<Note>
Permission changes take effect immediately for online players without a server restart. Use `/lp editor` for a browser-based GUI to manage permissions in bulk.
</Note>
